## SYNOPSIS

Dummy module to ensure PlatyPS generated parameter yaml blocks and named parameter set headings render as expected.

## SYNTAX

### ByName (Default)

```powershell
Test-ParameterYamlBlocks -Name <string> [-Force]
```

### ByLimit

```powershell
Test-ParameterYamlBlocks [-Limit <int>] [-Force]
```

## DESCRIPTION

Ensures that:

- the yaml code blocks are not touched by the markdown transformations
- the SYNTAX section keeps a heading for each named parameter set
- parameter descriptions are transformed like any other markdown (encoding
  raw &lt;brackets&gt; and \{curly\} brackets but not `<inline code>`)

## PARAMETERS

### -Force

A switch parameter available in all parameter sets.

```yaml
Type: System.Management.Automation.SwitchParameter
DefaultValue: False
SupportsWildcards: false
Aliases: []
ParameterSets:
- Name: (All)
  Position: Named
  IsRequired: false
  ValueFromPipeline: false
  ValueFromPipelineByPropertyName: false
  ValueFromRemainingArguments: false
DontShow: false
AcceptedValues: []
HelpMessage: ''
```

### -Limit

An integer parameter with a default value.

```yaml
Type: System.Int32
DefaultValue: 10
SupportsWildcards: false
Aliases: []
ParameterSets:

  Position: Named
  IsRequired: false
  ValueFromPipeline: false

```
